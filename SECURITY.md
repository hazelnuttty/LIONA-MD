# Security Policy

## Supported Versions

We only support the latest major version of Liona-MD. Please ensure you are using the most recent release before reporting a vulnerability.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of Liona-MD seriously. If you discover a security vulnerability, please follow these steps:

1.  **Do Not Open a Public Issue**: To protect users, please do not report security vulnerabilities through public GitHub issues or public Telegram groups.
2.  **Contact the Maintainer**: Please report the vulnerability directly via Telegram to the developer: **[@hazeloffc](https://t.me/hazeloffc)**.
3.  **Include Details**: In your report, please include:
    * A description of the vulnerability.
    * Steps to reproduce the issue.
    * Potential impact of the vulnerability.

### Response Timeline

* We will acknowledge your report within 48 hours.
* We will provide an estimated timeline for the fix.
* Once fixed, we will release a patch and credit you for the discovery (unless you prefer to remain anonymous).

## User Responsibility regarding Tokens

Liona-MD is a self-hosted framework. As a user/developer, you are responsible for keeping your **Telegram Bot Token** and **Session Data** secure.
* Never commit your `config.js` or `sessions/` folder to a public repository.
* If you suspect your token has been compromised, revoke it immediately via BotFather.

## Scraping Ethics

This bot contains scraping features (`/scrape`). Users are responsible for ensuring their usage complies with the Terms of Service of the target websites. Liona-MD is not responsible for IP bans or legal issues arising from aggressive scraping.
